Why use Nijor ?

Reusability

Write your code in reausable components which can be reused multiple times, thus making your code look cleaner, easier to debug and more organized. Write components in *.nijor files.

Scoped CSS

CSS inside your components are scoped which means that you don't need to think about your component's css affecting other components or vise-versa. You can write global styles in the style.css file.

Routing

A file-system based client-side router. Works automatically based on your file arrangement. No configuration required ; no extra code needed. Supports parameterized routing.

SSR Support

SSR in Nijor works a little different than other frameworks. However, it doesn't effect SEO adversely.
