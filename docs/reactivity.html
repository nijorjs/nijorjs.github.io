Reactivity

Reactive variables in Nijor provide a powerful way to manupulate the DOM with less code !
Note : Reactivity is still in development phase. Hence, thus feature might be a little buggy in 1% of the cases.

Update UI

Reactive variables update the UI.
Let's learn using a simple example. 


            <body>
                <h1>Counter</h1>
                <p>Count is {@count}</p> <!-- This tag gets re-rendered whenever the value of @count changes -->
                <p>Double is {@double}</p> <!-- This tag gets re-rendered whenever the value of @double changes -->
                <div>
                    <button on:click="Decrement()">-</button>
                    <button on:click="ResetCount()">Reset</button>
                    <button on:click="Increment()">+</button>
                </div>
            </body>

            <script>
                // Variables of type @varname are reactive variables. Reactive variables can only be declared inside a .nijor file, not inside a .js file
                
                const @count = 0; // Reactive variables are declared using the const keyword, but their value can be changed.
                const @double = @count * 2; // It's a derived reactive variable ; it's value changes whenever the value of @count changes.
                
                function Increment() {
                    @count++;
                }
                function Decrement() {
                    @count--;
                }

                function ResetCount() {
                    @count = 0;
                }
            </script>

Gathering input data

n:ref attribute is used for one-way data-bindings of a reactive variable to an input field. Whenever the value of the input field changes, the value of the reactive variable changes, but not the other way round. 


            <body>
                <input n:ref="@name" type="text" placeholder="Type your name ..."/> <!-- Whenever the value of this input changes, @name will change -->
                <p>Hello {@name}</p> <!-- As you type in the input box, the text inside this tag will get updated in real time -->
            </body>
            <script>
                const @name = '';
            </script>
n:ref is useful for gathering data from inputs for form submitions. 

            <body>
                <input n:ref="@name" type="text" placeholder="Type your name ..."/>
                <input n:ref="@age" type="number" placeholder="Type your age ..."/>
                <button on:click="sendData()">Submit</button>
            </body>
            <script>
                const @name = '';
                const @age = 0;

                function sendData(){
                    let data = {name: @name, age: @age};
                    // fetch(....);
                }
            </script>

Subscribe to events

You can subscribe to reactive variables. Whenever a reactive variable changes it's value, the specified function will run. 


            const @msg = 'Hello';
            @msg.subscribe(value=>console.log(value)); // Whenever @msg changes, the new value of @msg will be printed to the console.
            @msg = 'Hi'; // 'Hi' gets printed to the console.

Importing/Exporting Reactive Variables

You can import reactive variables from other components to change the UI in that other component.

File : src/components/msgbar.nijor


            <body>
                <div>
                    <p>Msg is : {@msg}</p>
                </div>
            </body>
            <script>
                const @msg = 'Hello';
                export { @msg };
            </script>

File : src/pages/input.nijor


            <body>
                <div>
                    <input type="text" n:ref="@msg"/>
                </div>
            </body>
            <script>
                import { @msg } from 'components/msgbar.nijor';
            </script>
